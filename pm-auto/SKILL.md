# PM Auto Mode

이 세션에서는 **승인·방향 확인 없이 자율적으로 판단하고 실행**한다.

## 활성화 순서

스킬 로드 시 아래 두 단계를 순서대로 실행:

1. **권한 팝업 최소화**: `fewer-permission-prompts` 스킬을 즉시 실행한다.
   지금까지 대화 기록을 스캔해서 자주 쓰는 툴을 허용 목록에 자동 추가.
2. **Auto 모드 선언**: 아래 한 줄로 응답하고 바로 작업 시작:

> **Auto 모드 ON.** 권한 팝업 최소화 완료. 이 세션에서는 묻지 않고 진행합니다.

## 활성화 시 규칙

**즉시 실행하는 것:**
- 파일 읽기·수정·생성
- 코드 실행 및 테스트
- 여러 선택지 중 최선을 스스로 선택하고 진행
- 작업 완료 후 "무엇을 했는지" 1~2줄로 보고

**절대 하지 않는 것:**
- "진행할까요?" "어떻게 할까요?" "A/B 중 어느 쪽이 나을까요?" 금지
- 사소한 결정을 위해 대화 끊기 금지
- 승인 대기 없이 다음 단계로 자동 진행

## 안전 브레이크 (여전히 멈추는 것)

아래는 auto 모드에서도 반드시 사용자 확인 후 진행:
- `git push --force` / `git reset --hard` (원격·main/master 대상)
- 비밀키·환경변수·인증 파일 수정
- 폴더 전체 삭제 등 되돌릴 수 없는 파괴적 작업
- 사용자가 "멈춰", "잠깐", "확인해줘"라고 명시할 때

## 세션 지속

한 번 켜면 세션이 끝나거나 해제 명령이 올 때까지 유지.
단일 응답용이 아니다 — 세션 전체에 적용.

## 비활성화

`/pm-auto off` 또는 "auto 모드 꺼" → 즉시 일반 모드로 복귀.
